GMAIL AUTOMATOR

Schedule and automate Gmail sends with precise timing. Connect any Google account via OAuth, write your schedule in a simple text file, and let the server fire emails on time — daily or on repeating intervals.

ℹ
Server-side tool. Unlike the other tools here, Gmail Automator requires a backend to handle OAuth and scheduling. It's deployed separately on Render. Click Launch to open the live app, or follow the Setup guide below to self-host it.
Launch App Source on GitHub
features
🔐
Password Protected
App-level password gate keeps the scheduler private and accessible only to you.
🔗
Gmail OAuth2
Any Google account can connect. Only send access is requested — no reading of emails.
⏰
Daily Scheduling
Fire an email at a specific time every day using simple HH:MM notation.
🔁
Interval Sending
Repeat an email every N minutes with every:10m — perfect for reminders (or chaos).
📄
File-based Schedule
Write all your scheduled emails in a plain .txt file and bulk-import them in one click.
✍️
Manual Compose
Quick one-off sends or scheduled messages directly from the dashboard UI.
schedule file format
# One email per line. Pipe-separated.
# Format: TIME | RECIPIENT | SUBJECT | BODY

# Daily at HH:MM
09:00 | friend@gmail.com | Morning! | Have an amazing day 🌅
20:30 | mum@gmail.com | Goodnight | Love you, sleep well ✨

# Repeating every X minutes
every:10m | ex@gmail.com | Hey... | I miss you, please respond 😭
FieldFormatExample
TIMEHH:MM for daily · every:Xm for interval09:00 · every:10m
RECIPIENTValid email addresssomeone@gmail.com
SUBJECTPlain textGood morning!
BODYPlain text (emoji OK)Have a great day 🌟
self-hosting setup
1
Clone the repository
git clone https://github.com/tibetthetibz-alt/gmail-automator
cd gmail-automator && npm install
2
Create a Google OAuth app
Go to Google Cloud Console → New project → APIs & Services → OAuth consent screen → Add scope gmail.send.
Then Credentials → Create OAuth 2.0 Client ID (Web app) → add your Render URL as an authorised redirect URI: https://your-app.onrender.com/auth/callback
3
Configure environment variables
GOOGLE_CLIENT_ID=your_client_id
GOOGLE_CLIENT_SECRET=your_client_secret
REDIRECT_URI=https://your-app.onrender.com/auth/callback
SESSION_SECRET=any_long_random_string
4
Deploy to Render
Connect your fork on render.com → New Web Service → Build: npm install · Start: npm start · Add env vars above. A render.yaml is included in the repo for one-click deploy.
